Berechnen und Anzeigen
von SLD-Bäumen

Mit dem Web-Interface können Sie SLD-Bäume generieren und anzeigen lassen.
Die SLD-Bäume werden am Server von SWI-Prolog berechnet und im Browser-Fenster mittels JavaScript/processing.js angezeigt.

Um dieses Dokument in einem neuen Fenster zu öffnen klicken Sie bitte hier: info.html. (Wenn Sie das Dokument außerhalb dieses Frames anzeigen funktionieren allerdings eine Links zu JavaScript-Funktionen nicht.)

Sollten Sie weitergehende Fragen haben, kontaktieren Sie mich bitte hier.

    • Bemerkungen und Einschränkungen

    Antwort-Substitution

    Die korrekte Antwort-Substitution eines erfolgreichen Astes wird durch Doppelklick auf den Blattknoten angezeigt. Beim Doppelklick auf einen inneren Knoten, sowie beim Doppelklick auf einen Blattknoten eines erfolglosen Astes, wird als Antwort ebenfalls die Substitution (beschränkt auf die Eingabevariablen) angezeigt. Im Falle eines inneren Knotens handelt es sich dabei um die bis dahin berechnete (möglicherweise unvollständige) Antwort. Im Fall eines erfolglosen Astes wird ebenfalls die entsprechende Substitution als Antwort angezeigt. Dabei handelt es sich allerdings um keine gültige Antwort.

    Knoten Limit

    Die Anzahl der Knoten, die in einem Berechnungsshritt generiert werden ist auf 1000 beschränkt. Sie können einen Baum mit mehr als 1000 Knoten berechnen indem Sie den Baum schrittweise aufbauen. Falls zu einem Knoten nicht alle Kinder erzeugt werden, kann es notwendig sein die Anzahl Levels, die in einem Schritt berechnet werden zu reduzieren.

    Cuts

    Cuts werden nicht berücksichtigt. Es wird immer der ganze Baum (unter Berücksichtigung der maximalen Anzahl an Levels und Knoten) berechnet.

    Das If->Then;Else Konstrukt wird interpretiert als If,!,Then;Else. Dies ist nicht korrekt weil im Else-Fall ebenfalls ein Cut erfolgen sollte. Da Cuts jedoch nicht berücksichtigt werden, wird der Baum unabhängig von If immer für den Then- und ebenso für den Else-Fall berechnet. Dadurch kann ein Ast fälschlicherweise mit 'true' enden und eine falsche Antwort-Substitution liefern.

    Eingebaute Prädikate

    Eingebaute Prädikate sind nur teilweise verfügbar.

    Anstelle der Klausel-Nummer wird für eingebaute Prädikate die Nummer 0 in der Graphik angezeigt.

    Verfübar sind:
    ';', ',', '=', is, Term-Vergleiche, Arithmetische Vergleiche, ...

    Nicht verfügbar sind:
    Meta-Prädikate, Ein-/Ausgabe, ...

    Arithmetishe Funktionen können verwenet werden.

    Listen können verwendet werden in [ ]-Schreibweise. Der '.'-Operator kann nicht verwendet werden.
